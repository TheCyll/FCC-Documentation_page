Comments

Comments are a good practice in coding. They help us to:

  • Order our ideas
  • Understand the function of line of code
  • Communicate our ideas to other programmers
  • Give structure to our programm

We can put comments in our programm in two ways:

//this is a comment

This type of comment is for only 1 line of text. All text int that line is a comment

/* this is a multi-line comment */

This other type is for comments in more than one line. For example:

/*this
is
a multi-line
comment */
Class declaration

For the start we need to declarate a class. We can see this in line 3 of our code. All programm has a class that you have to define. The 'class' keyword declarates a class and the name of this class must come inmediately

We can name this class with and identifier. An identifier is a way to write names that have some characteristics.

In programming we commonly use names for identification of variables that are clear and well notated. We use camel-case, meaning to intercalate Low and upper case like this examples:

  • ThisIsCamelCase
  • NiceIdentifierName
  • VariableSuperDuperTruper

We can also use numbers in our identifiers. A lot of IDE only support numbers at the end of the identifier, NEVER in the beggining. For example:

  • VariableX1
  • NameOfTeacher2
  • CommonCamelCaseFor4
Method declaration

The 6 line

public static void main( String[] args)

is a method declaration. This is the start point of any Java application. The parenthesis after the 'main' word indicate that this is a construction block of the programm, which is called method. Java declarations often have one or more methods. In a Java application, just one of this methods must be main and it has to be defined as the 6 line. The keyword void means this method won't return any information. The words String[] args between parenthesis are a part of the main method declaration.

The left key in line 7 beggins the body of the method declaration. Its corresponding right key must end it in line 9.

String []args
Output operations

The line 8

System.out.println( "Welcome to Java coding!);

tells the computer to make a computation; meaning to print a string of characters in between the double quotes (without include them).

System.out is known as the standard output object. It enables the Java aplications to show information in the comand window.

The method System.out.println shows (prints) a tex line in the comand window. The string in the parenthesis in line 8 is the argument for the method.
